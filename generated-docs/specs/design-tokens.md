# RBAC Demo — Style Guide

> Generated by design-style-agent | Source: design-tokens.css

## Design Principles

This is a functional demo application. The design prioritises:

1. **Clarity** — role-based access rules must be immediately obvious to anyone reviewing the demo
2. **Accessibility** — WCAG AA contrast throughout; all interactive elements keyboard-operable
3. **Simplicity** — no decorative elements; clean Tailwind + Shadcn defaults

## Spacing

Use the 4px grid (Tailwind spacing scale). Key patterns:

- Component internal padding: `p-4` (16px) or `p-6` (24px)
- Page horizontal padding: `px-4 sm:px-6 lg:px-8`
- Section gaps: `gap-6` (24px)
- Narrow gaps (inline): `gap-2` (8px)

## Layout

### Navigation Bar

- Height: `3.5rem` (`--nav-height`)
- Background: `--card` with `border-b`
- Left: app name / nav links; Right: user display name + logout button
- Sticky at top (`sticky top-0 z-[100]`)

### Login Page

- Vertically and horizontally centred on viewport
- Card width: `28rem` (`--login-card-width`) max, `w-full` on mobile
- Card: `<Card>` from Shadcn with `shadow-md`

### Content Pages (Page 1, 2, 3)

- Max width: `--max-content-width` (1280px), centred with `mx-auto`
- Top padding: `pt-12` to clear the nav bar

## Component Patterns

### Login Form

```tsx
<Card className="w-full max-w-[28rem] shadow-md">
  <CardHeader>
    <CardTitle>Sign in</CardTitle>
  </CardHeader>
  <CardContent>
    <form className="flex flex-col gap-4">
      <div className="flex flex-col gap-1.5">
        <Label htmlFor="username">Username</Label>
        <Input id="username" name="username" autoComplete="username" />
      </div>
      <div className="flex flex-col gap-1.5">
        <Label htmlFor="password">Password</Label>
        <Input id="password" name="password" type="password" autoComplete="current-password" />
      </div>
      {/* Error state — only render when error exists */}
      <p className="text-sm text-destructive" role="alert">Invalid credentials</p>
      <Button type="submit" className="w-full">Sign in</Button>
    </form>
  </CardContent>
</Card>
```

### Navigation Bar


## Accessibility

- All form inputs paired with `<Label htmlFor="...">`
- Error messages use `role="alert"` for live region announcement
- Focus-visible styles: Tailwind `focus-visible:ring-2 focus-visible:ring-ring` (from Shadcn defaults)
- Minimum touch target: 44×44px for all interactive elements
- Colour contrast: blue-600 on white = 4.6:1 (exceeds WCAG AA 4.5:1)
